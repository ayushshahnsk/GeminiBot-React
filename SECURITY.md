## Reporting a Vulnerability

If you discover a security vulnerability in **GeminiBot-React**, please report
it responsibly.

### How to Report
- **Do not** open a public GitHub issue for security vulnerabilities.
- Instead, contact the project maintainer directly.

You may report security issues by:
- Opening a **private security advisory** via GitHub (preferred), or
- Contacting the repository owner through GitHub.

Please include:
- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact (if known)
- Any suggested fixes or mitigations

---

## Response Timeline

The maintainer aims to:
- Acknowledge valid security reports within **72 hours**
- Assess and address confirmed vulnerabilities as quickly as possible

Timelines may vary depending on the severity and complexity of the issue.

---

## Disclosure Policy

- Security vulnerabilities will be fixed before public disclosure whenever possible.
- Credit will be given to reporters who responsibly disclose vulnerabilities,
  unless anonymity is requested.

---

## Security Best Practices for Contributors

Contributors are encouraged to:
- Avoid committing secrets (API keys, tokens, passwords)
- Use environment variables via `.env` files
- Follow secure coding practices
- Keep dependencies up to date
